# WSL 下批量压缩图像（减小体积、保持文件名）

用于在上传 COLMAP / OpenSplat 工程前缩小 `images` 文件夹占用。**不抽帧**：张数与文件名（含扩展名）与输入一致，便于继续配合已有 `sparse`（前提：建稀疏模型时用的就是这一套路径与命名）。

---

## 1. 环境

在 WSL（Ubuntu）终端执行：

```bash
sudo apt update
sudo apt install -y python3 python3-pip
pip3 install --user pillow
```

验证：

```bash
python3 -c "from PIL import Image; print('OK')"
```

---

## 2. 脚本位置与作用

| 项目 | 说明 |
|------|------|
| 脚本路径 | `colmap_ws/compress_images.py` |
| 工程根目录 | 脚本内 `BASE = Path("/home/ccxx/colmap_ws")` |
| 默认输入 | `BASE / "images"` |
| 默认输出 | `BASE / "images_small"`（自动创建） |
| JPEG | 指定质量重编码（默认 `quality=82`） |
| PNG | 无损优化 + 最高 zlib 等级（体积略降；若仍过大需另考虑 `pngquant` 或改流程） |

**请勿**把 Python 源码整段粘贴进终端执行；应保存为 `.py` 后使用 `python3` 运行（见下）。

---

## 3. 修改路径（必看）

用编辑器打开 `compress_images.py`，改 **`BASE`**（以及需要时的 **`SRC` / `DST`**）：

```python
BASE = Path("/home/ccxx/colmap_ws")
SRC = BASE / "images"
DST = BASE / "images_small"
```

若数据在 **Windows D 盘**，例如：

```python
BASE = Path("/mnt/d/software/colmap")
SRC = BASE / "images"
DST = BASE / "images_small"
```

若只对 **`images_small` 做第二轮压缩**，按脚本内注释改为 `SRC = BASE / "images_small"`、`DST = BASE / "images_small2"` 等。

可调参数：

```python
JPEG_QUALITY = 82   # 1–95，越小文件越小，建议先 82–88
PNG_COMPRESS = 9    # PNG zlib 0–9
```

---

## 4. 运行

```bash
cd /home/ccxx/colmap_ws
python3 compress_images.py
```

看到每张图输出 `文件名 -> xxx KB`，最后打印 `Done.` 即完成。

**两行命令要分开执行**（不要写在同一行用 `#` 注释掉第二行导致没执行）：

```bash
pip3 install --user pillow
python3 /home/ccxx/colmap_ws/compress_images.py
```

---

## 5. 与 COLMAP / OpenSplat 的兼容性

1. **不要改分辨率**（脚本默认不改宽高），否则与相机标定/稀疏模型可能不一致。  
2. **不要改文件名与扩展名**；若原图为 `.png`，输出仍为 `.png`。  
3. **JPEG 另存为 JPEG**、**PNG 另存为 PNG**，与稀疏重建时一致即可。  
4. 若你曾用 **另一套图名** 建过 `sparse`，替换为 `images_small` 后需保证 **与 `images.bin` 里记录的名称一致**（通常同一工程内仅换更小文件即可）。

检查体积：

```bash
du -sh images images_small
```

确认无误后，可用 `images_small` 与 `sparse/0` 一起打包上传。

---

## 6. 有损压缩 PNG（pngquant，体积明显下降）

**特点**：仍是 **`.png` 文件名**，尺寸（宽×高）不变，一般可与现有 `sparse` 配合使用；属于**有损**，画质会略降。

### 6.1 安装

```bash
sudo apt update
sudo apt install -y pngquant
```

### 6.2 一键脚本（推荐）

工程目录下的 **`pngquant_images.sh`** 会：删除旧的 `images_pngquant`（若存在）→ 从 `images_small` 复制 → 逐张 `pngquant` 并打印 **`[当前/总数] 文件名`**。

```bash
cd /home/ccxx/colmap_ws
chmod +x pngquant_images.sh
./pngquant_images.sh
```

可选环境变量（默认值见脚本内）：

```bash
BASE=/home/ccxx/colmap_ws SRC_NAME=images_small DST_NAME=images_pngquant COLORS=256 ./pngquant_images.sh
```

- **`COLORS`**：调色板颜色数，`128` 更小、`320` 略更清晰。

### 6.3 手动命令（与脚本等价）

```bash
cd /home/ccxx/colmap_ws

cp -a images_small images_pngquant

cd images_pngquant
for f in *.png; do
  [ -f "$f" ] || continue
  pngquant --ext .png --force 256 "$f"
done
cd ..
du -sh images_small images_pngquant
```

### 6.4 检查体积与效果

```bash
du -sh /home/ccxx/colmap_ws/images_pngquant
```

随机打开几张图看是否可接受；**上传/给 OpenSplat 用时**，把工程里的 **`images` 换成 `images_pngquant` 的内容**（文件夹名仍建议对外叫 **`images`**，与 `sparse` 里路径一致）。

---

## 7. 可选：仅 JPEG 时用系统工具（原地压缩，先备份）

```bash
sudo apt install -y jpegoptim
# 会覆盖原文件，请先复制整个 images 文件夹再操作
jpegoptim --max=85 --strip-all /path/to/images/*.jpg
```

---

## 8. 常见问题

| 现象 | 处理 |
|------|------|
| `No module named 'PIL'` | 执行 `pip3 install --user pillow` |
| 粘贴脚本到终端报 `import: command not found` | 必须用 `python3 compress_images.py` 运行文件 |
| PNG 体积几乎不变 | PNG 无损优化幅度有限；可略降 `JPEG_QUALITY` 若原图实为 JPEG 扩展名错误等 |

---

脚本全文见同目录 **`compress_images.py`**；修改路径后以 **`python3 compress_images.py`** 执行为准。
